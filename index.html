Forms are basically elements within a page where you can collect data and transport it to a specified website that you choose, which is under the action part. Descendant selectors

Within html elements, There can also be descendant selectors where you can include specific elements within another element. For example, you can add a hyperlink to a paragraph element. It is also noted that it is a best practice that you should only have 3 descendant selectors.

CSS grouping

Now moving on to the css section of this week's notes. So starting right off, I have learnt that you can group elements together in order to style them together in the css section. What do i mean by that is that you can style both a paragraph or section element at the same time in the css file by just including a comma. Next, I have learnt more about precendence order in which a css file would read the codes. Certain elements would have more specificity than others. For example, a class element would take precendece over a paragraph. Meaning that if both elements were styled and used in a element in the html code, the class styling would be carried out instead of the paragraph one. However, you can eradicate this rule if you just add the !important to the element in the css file. This allows the specific element to take precedence over all others, regardless of the previous rule that I had just stated.

Media Queries

Media Queries allows you to specify the CSS styling in accordance to the device a user is using. For example, you can @media and (width: 576px). This code allows you to edit how the webpage would look like to mobile users. Furthermore, you can also specify the media types, for example : screens.

CSS Positioning

Positioning allows you to position your elements however you would want them to do so. In this weeks notes, I have learnt about 3 of them. Those are absolute,relative and fixed. So starting right off, absolute positioning makes a element's position relative to a parent element that has a absolute positioning. If no parent element has absolute positioning, the element would just be relative to the browser screen. Meaning that if lessay you add top: 0px;, left: 0px; and position: absolute; to a box, the box would be positioned at the top left hand side of a browser. Moving on, there is relative. This positioning allows you to alter the positioning of an element in relative to its original positioning. Next, there is the fixed positioning. It is similiar to absolute but unlike it, it will always be relative to the window's browser instead of a parent element and it is also a sticky element, where it does not scroll with the rest of the page. Moving on, there is also a z-index which helps to specify which elements are on top of one another, something like stacking (gentle note: z-index only works on elements wit a specified positioning like the 3 ones i stated above). A negative z-index would mean the element would be at the bottom and a positive one would mean the element would be at the top.
