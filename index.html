CSS Positioning

Positioning allows you to position your elements however you would want them to do so. In this weeks notes, I have learnt about 3 of them. Those are absolute,relative and fixed. So starting right off, absolute positioning makes a element's position relative to a parent element that has a absolute positioning. If no parent element has absolute positioning, the element would just be relative to the browser screen. Meaning that if lessay you add top: 0px;, left: 0px; and position: absolute; to a box, the box would be positioned at the top left hand side of a browser. Moving on, there is relative. This positioning allows you to alter the positioning of an element in relative to its original positioning. Next, there is the fixed positioning. It is similiar to absolute but unlike it, it will always be relative to the window's browser instead of a parent element and it is also a sticky element, where it does not scroll with the rest of the page. Moving on, there is also a z-index which helps to specify which elements are on top of one another, something like stacking (gentle note: z-index only works on elements wit a specified positioning like the 3 ones i stated above). A negative z-index would mean the element would be at the bottom and a positive one would mean the element would be at the top.

Local Storage

Most browser store up to 5MB of data per domain in a storage object and if a site requires more than 5MB of data, the broswer will typically ask the user for permission to store data(never rely on users agreeing to give a site more space). “QUOTA_EXCEEDED_ERR” is the exception that will get thrown if you exceed your storage quota of 5 megabytes. You do try and catch if you wana do something when the website reaches this error. E.g.

try {

localStorage.setItem(data.name,JSON.stringify(data));

} catch(domException) {

if (domException.name === 'QuotaExceededError' ||

domException.name === 'NS_ERROR_DOM_QUOTA_REACHED')

{

// Fallback code comes here.

}

}

Local storages are based on key/values pairs(Something like dictionaries in python and C#). There are 2 main types of local storages, Local and session. They both have the same methods to get and set data, but each lasts a different amount of time. For security, browsers employ a same origin policy, which means that data can only be accessed by other pages in the same origin. Some examples would include that, Protocol, Subdomain, Domain, Port must match. So how do you start with local storage, you start with this: localStorage.setItem(key, value) or sessionStorage.setItem(key, value) and to get the items, you do this: localStorage.getItem(key) or sessionStorage.getItem(key).Another thing to note is that you can treat the storage as an array meaning, you can get an item like this localStorage["name of key"].

Similiarly, it is made up of keys and values pairs.The values can be a string, number, Boolean, array, object or null. So in order to utilise json well, javascript has 2 important methods on how to manipulate json to help you out.

  • Convert a JavaScript object to a string: -JSON.stringify();
  • Convert a string to a JavaScript object: -JSON.parse();
